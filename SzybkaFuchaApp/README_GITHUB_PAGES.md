# GitHub Pages Setup

## Target state

The blog should be published by GitHub Actions from:

- source: `/Users/simacbook/.openclaw/workspace/SzybkaFuchaApp/blog`
- artifact: `/Users/simacbook/.openclaw/workspace/SzybkaFuchaApp/blog/public`

GitHub Pages setting must be:

- `Settings -> Pages -> Build and deployment -> Source: GitHub Actions`

## What GitHub Pages serves

After the workflow runs, the live site is built from the uploaded artifact generated by:

```bash
cd /Users/simacbook/.openclaw/workspace/SzybkaFuchaApp/blog
npm run build
```

GitHub Pages does not need repo-root `index.html`, `blog/*.html`, or `images/*` anymore.

## Local publishing command

```bash
bash /Users/simacbook/.openclaw/workspace/SzybkaFuchaApp/blog/scripts/daily-post.sh
```

This command:

1. runs the local build
2. stages only `SzybkaFuchaApp`
3. commits the changes
4. pushes to `master`
5. lets GitHub Actions deploy Pages
